## 3. <span style="color:#2580f7">Sourcetree</span>

### Was ist Versionskontrolle?
Versionskontrolle ist der Prozess des Verfolgens und Verwalten von Änderungen an Dateien im Laufe der Zeit. Es ermöglicht Ihnen, ältere Versionen Ihrer Dateien wiederherzustellen, Änderungen zu vergleichen und zu sehen, wer Änderungen vorgenommen hat.

### Was ist SourceTree?
SourceTree ist ein Tool, welches ein grafisches Nutzerinterface dafür bereitstellt und die Versionskontrolle so erleichtert.

## 6. Projekt auschecken

.... Beudetet?

Wenn man bei GitHub ein Projekt auscheckt, dann lädt man eine Kopie des Projekts von GitHub auf seinen eigenen Computer herunter, um daran zu arbeiten. Das Auschecken ist der erste Schritt, um auf einem Projekt zu arbeiten, das auf GitHub gehostet wird. Es ermöglicht einem, das Projekt lokal auf seinem eigenen Computer zu bearbeiten, ohne dass man das Originalprojekt auf GitHub verändert. Nachdem man seine Änderungen vorgenommen hat, kann man sie wieder zurück auf GitHub hochladen (was wir in deinem Fall aber nicht machen werden, da das Projekt nur für dich allein ist), um sie mit anderen zu teilen oder um eine Zusammenarbeit zu ermöglichen. Um ein Projekt mithilfe von SourceTree von GitHub zu klonen folge diesen Schritten:

- [ ] Schritt 1: Öffne SourceTree und klicke auf File -> Clone / new (bzw. Datei -> Klonen / Neu)

- [ ] Schritt 2: Öffne das Git-Repository, das du auschecken möchtest, in deinem Browser - in diesem Fall das Schuelerpraktikum-Repository auf GitHub. Klicke auf den grünen Button "Code" und wähle HTTPS. Kopiere die HTTPS-URL des Repositories.

- [ ] Schritt 3: Füge die Repository-URL in SourceTree ein. Gehe zurück zu SourceTree und füge die kopierte HTTPS-URL in das Feld "Source URL" bzw. "Quellpfad / URL" im Clone-Dialog ein. Wähle den Ordner, in den das Repository ausgecheckt werden soll (an dieser Stelle erstellst du am besten einen extra Ordner dafür), und gib einen Namen für das Repository ein.

- [ ] Schritt 4: Klicke auf "Clone" im unteren Bereich des Clone-Dialogs. SourceTree wird das Git-Repository auschecken und auf deinem Computer speichern. Nachdem der Clone-Vorgang abgeschlossen ist, wird das Repository in SourceTree angezeigt. Klicke auf den Namen des Repositorys, um es zu öffnen.

## 7. Projekt lokal öffnen

Nachdem du das **Projekt erfolgreich geklont** und somit eine **lokale Kopie** davon in einem Ordner auf deinem Laptop vorliegen hast, kannst du das Projekt jetzt in deinem **Code Editor** (Visual Studio Code) öffnen. Dafür öffnest du Visual Studio Code, und öffnest den Ordner in welchem du das Projekt abgelegt hast: File -> Open Folder. Falls ein Dialog kommt kannst du "trust the authors" klicken, das Programm möchte nur sicher gehen, dass das Projekt "sicher" ist.

Falls du alles richtig gemacht hast, sollte das Projekt jetzt in deinem Code-Editor angezeigt werden. Du hast jetzt die Möglichkeit, die weiteren **Texte entweder weiterhin auf GitHub zu lesen** oder direkt in Visual Studio Code. Solltest du dich für VSCode entscheiden, musst du auf die entsprechenden Markdown-Dateien (die mit der Endung ".md") rechtsklicken und "open preview" auswählen, da die Dateien ansonsten ziemlich unschön zu lesen sind.

## 8. Trello
Trello ist ein auf Kanban basierender Aufgaben-Verwaltungs-Onlinedienst des Herstellers Atlassian. Es wird dafür genutzt um Teamarbeit effizienter zu gestalten und Projekte leicht zu organisieren.

### Kanban Board
Um dir den Beruf eines Entwicklers so nah wie möglich zu bringen haben wir dir ein **Kanban Board** erstellt, da ein solches Board ein fester Bestandteil unseres Arbeitsalltags ist. Du hast eine **Übersicht der Tasks**, siehst **Fortschritte** und den aktuellen **Bearbeitungsstatus**.
Wenn dich interessiert wie ein solches Board in Kundenprojekten aussiehst, komm gerne auf uns zu.

### Was bedeuten die verschiedenen Spalten/Bereiche?
- Die Karten bzw. Aufgaben können zu den verschiedenen Bereichen verschoben werden. Am Anfang des Tages kann die Spalte **"To Do"** gut dafür genutzt werden, sich Ziele zu setzen und sich den nächsten Schritten bewusst werden.

- Aktuell bearbeitete Aufgaben können im Bereich **"Im Gange"** liegen, damit man auch nach dem Wochenende noch weiß, wo man war.

- Eine abgeschlossene Aufgabe kann man nach **"Fertig"** verschieben.


### Nutzung des Ticketboards
Das Board kannst du nutzen und anpassen wie du magst. Die Aufgaben dienen als Hilfestellung und sind daher kein Muss. Du kannst das Board um eigene Aufgaben ergänzen, wenn dir das bei der Planung und Strukturierung deiner Arbeit hilft.

Ein paar Beispiele für selbst erstellte Tickets:
- "Tutorial xy durcharbeiten"
- "Grundgerüst der Webseite erstellen"
- "Landingpage hinzufügen"
- "Navigationsleiste erstellen"
- "Hintergrundbild einfügen"
- "Schriftart ändern"
- ...

## 9. <span style="color:#6db33f">Backend — Spring Boot mit Kotlin</span>

[![My Skills](https://skillicons.dev/icons?i=spring,kotlin&theme=light)](https://spring.io/projects/spring-boot)

Im späteren Verlauf des Praktikums wirst du auch am Backend arbeiten. Das Backend ist der Teil einer Anwendung, der im Hintergrund läuft — er verarbeitet Daten, speichert sie in einer Datenbank und stellt sie dem Frontend über eine Schnittstelle (API) zur Verfügung.

### Was wird verwendet?
- **Spring Boot** — Ein Framework, das es einfach macht, Backend-Anwendungen in Java oder Kotlin zu erstellen. Es nimmt dir viel Konfigurationsarbeit ab.
- **Kotlin** — Eine moderne Programmiersprache, die auf der Java Virtual Machine (JVM) läuft. Sie ist kürzer und übersichtlicher als Java, aber genauso leistungsfähig.
- **MariaDB** — Eine relationale Datenbank, in der die Daten gespeichert werden. Sie läuft über Docker, damit du sie nicht selbst installieren musst.

### Wie ist das Backend aufgebaut?
Im Ordner `backend/demo/` findest du ein fertiges Beispielprojekt, das dir als Vorlage dient. Es zeigt das grundlegende Muster, nach dem du deine eigenen Endpunkte bauen kannst:

1. **Entity** — Definiert, wie ein Datensatz in der Datenbank aussieht (z.B. welche Felder eine Tabelle hat).
2. **Repository** — Stellt die Verbindung zur Datenbank her und ermöglicht es, Daten zu lesen und zu schreiben.
3. **Controller** — Definiert die API-Endpunkte, über die das Frontend Daten abrufen oder senden kann.

Das Demo-Projekt stellt z.B. den Endpunkt `GET /api/demo` bereit, über den du Testdaten abrufen kannst.

### Was muss ich installieren?
- **Docker Desktop** — Wird benötigt, um die MariaDB-Datenbank zu starten. Dein Betreuer hilft dir bei der Einrichtung.
- **IntelliJ IDEA** (empfohlen) — Eine Entwicklungsumgebung, die besonders gut mit Kotlin und Spring Boot funktioniert. Du kannst aber auch VSCode verwenden.

### Wie starte ich das Backend?
1. Starte Docker Desktop und führe `docker compose up` im Ordner `backend/demo/` aus — das startet die Datenbank.
2. Starte die Spring Boot Anwendung (in IntelliJ: einfach den grünen Play-Button drücken).
3. Das Backend läuft dann auf `http://localhost:9090`. Probier es aus: öffne `http://localhost:9090/api/demo` im Browser.

### Wie verbinde ich Frontend und Backend?
In der Datei `frontend/React-TypeScript-Projekt/frontend/.env` findest du die Variable `VITE_API_URL`. Wenn du mit dem Spring Boot Backend arbeitest, ändere den Port dort von `5000` auf `9090`.
